# Graph Contrastive Learning for Sheet Music-Audio Alignment

This repository implements a cross-modal retrieval system mapping piano sheet music lines directly to audio spectrograms. Instead of using standard modality-specific contrastive pretraining, such as contrasting augmented views of data snippets ([Chaudhuri et al., 2023](https://arxiv.org/pdf/2309.12134)), we utilize a Graph Attention Network (GAT) as a pretraining proxy. We construct a graph representation using annotations from the MSMD dataset ([Dorfer et al., 2018](https://transactions.ismir.net/articles/10.5334/tismir.12)). By aligning both the audio and vision encoders to the graph embeddings first, we establish a shared semantic space before directly fine-tuning the modalities against each other. 

## Methodology & Architecture
The graph representation is as:

**Node Features:**
Node features are extracted using annotations from the MSMD dataset. Each node represents a musical note and contains:
* **Continuous Features:** Spatial attributes derived from the sheet music image (bounding box top, left, width, and height) and temporal features from MIDI data (onset and duration).
* **Pitch:** MIDI pitch code

**Edge Construction:**
The adjacency matrix is constructed exclusively from temporal relationships:
* **Chords:** Fully connected subgraphs for notes sharing the exact same onset frame.
* **Sequences:** Forward-directed edges connecting all notes at time $t$ to all notes at time $t+1$.

We map individual sheet music systems to their corresponding audio spectrogram crops.

1. **Audio-Graph Pretraining:** We use symmetric Momentum Contrast (MoCo) to align audio spectrogram embeddings with their corresponding graph embeddings generated by the GAT.

3. **Vision-Audio Pretraining:** The pre-trained Vision and Audio encoders are jointly fine-tuned using symmetric MoCo to directly align the sheet music images with the audio spectrograms.
